# Exoplanets 6 — Porto 2026 · Conference Guide

A self-contained web app to navigate the EXO6 programme: a **week overview**, a
**by-day timetable** with parallel rooms side-by-side (so you can see your
afternoon options when sessions overlap), **search/filter** by topic / room /
theme, a **personal starred agenda** with clash detection, and **shared group
notes & photos** on every talk.

Built from the official `EXO6_program.pdf`.

---

## Files

| File | What it is |
|------|------------|
| `index.html` | The whole app (open it directly in a browser to use locally). |
| `data.js` | The schedule data (202 talks, 30 sessions). Generated by `build_data.py`. |
| `firebase-config.js` | Where you paste your Firebase config to turn on shared notes. |
| `build_data.py` | Regenerates `data.js` if the programme changes. Run `python3 build_data.py`. |

Everything except the shared-notes feature works with **zero setup** — just open
`index.html`.

## Deploy on GitHub Pages (recommended — free, shareable link)

1. Create a repo, e.g. `exo6-guide`, and add these files
   (`index.html`, `data.js`, `firebase-config.js`, optionally `build_data.py`/`README.md`).

   ```bash
   cd exo6-guide
   git init
   git add index.html data.js firebase-config.js build_data.py README.md
   git commit -m "EXO6 conference guide"
   git branch -M main
   git remote add origin https://github.com/<your-username>/exo6-guide.git
   git push -u origin main
   ```

2. On GitHub: **Settings → Pages → Build and deployment → Source: “Deploy from a
   branch”**, pick branch `main` and folder `/ (root)`, **Save**.

3. After ~1 minute your guide is live at
   `https://<your-username>.github.io/exo6-guide/` — share that link with your group.

> Alternative (no account/git): drag the folder onto **https://app.netlify.com/drop**
> for an instant public link.

## Turn on shared notes & photos (optional, ~5 min)

The notes feature lets everyone with the link read and add notes/photos on each
talk. It uses **Firebase Firestore** (Google's free tier — generous limits, no
credit card needed for this usage).

1. Go to **https://console.firebase.google.com** → **Add project** (any name,
   e.g. `exo6-guide`). You can disable Google Analytics.

2. In the project, click the **Web** icon (`</>`) to *“Add an app to get
   started.”* Give it a nickname, **Register app**. You'll see a
   `firebaseConfig = { ... }` snippet — **copy the object**.

3. Left sidebar → **Build → Firestore Database → Create database**.
   Choose a location, and start in **Test mode** (this allows open read/write —
   fine for a trusted conference group; see the note below to tighten it).

4. Open `firebase-config.js` and replace `null` with your copied object:

   ```js
   window.FIREBASE_CONFIG = {
     apiKey: "AIza....",
     authDomain: "exo6-guide.firebaseapp.com",
     projectId: "exo6-guide",
     storageBucket: "exo6-guide.appspot.com",
     messagingSenderId: "1234567890",
     appId: "1:1234567890:web:abcdef123456"
   };
   ```

5. Commit & push (or re-drop on Netlify). Open the **“Notes & sharing”** tab in
   the app — it should say *“Shared notes are live.”*

That's it. The first time someone writes a note they'll be asked to type their
name (stored only in their own browser); their name is tagged on their notes.
Photos are auto-compressed and stored inline, so no extra Storage setup is
needed.

### A note on Firestore "test mode"
Test mode allows anyone with the link to read/write the `notes` collection and
expires after 30 days. For a conference that's usually fine. To extend it,
edit **Firestore → Rules**, e.g. to keep it open until after the conference:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /notes/{doc} {
      allow read, write: if request.time < timestamp.date(2026, 7, 31);
    }
  }
}
```

## How the guide is organised

- **Topics** (official, colour-coded):
  1. 🔵 Present-day results — setting the stage
  2. 🟢 Detection of the lowest-mass / smaller planets — methods & instrumentation
  3. 🟠 Detailed characterisation — interiors, atmospheres, astrobiology
  4. 🟣 Astrophysical challenges — stellar characterisation, star-planet, stellar noise

- Each **session block** has a distilled one-line **theme** (e.g. *“JWST phase
  curves & resolved atmospheres”*, *“Taming the Sun: stellar variability”*) shown
  on the day/week views and in each talk's detail panel.

- **Tags** on each talk (JWST, RV/EPRV, Direct imaging, Sub-Neptunes, …) are
  auto-derived from the title for quick filtering.

Updating the programme: edit the data in `build_data.py`, run
`python3 build_data.py`, and redeploy.
